🚨 ปัญหาร้ายแรง (Critical Issues) 6 รายการ

CRITICAL ไม่มีระบบติดตามสถานะการจัดส่ง (Delivery Tracking)

การออกแบบปัจจุบันจบที่ "JV Admin อนุมัติ" แต่ไม่มี flow สำหรับการจัดเตรียมสินค้า, จัดส่ง, และการรับสินค้าที่โรงพยาบาล ทำให้โรงพยาบาลไม่สามารถตรวจสอบได้ว่ายาจะมาถึงเมื่อไหร่

CRITICAL ไม่มีระบบ Recommended Order จากข้อมูลซื้อในอดีต

ตามที่ระบุว่าจะได้รับ DATA การซื้อในอดีตของ ร.พ. แต่ไม่มี Feature สำหรับวิเคราะห์ข้อมูลเพื่อแนะนำการสั่งซื้อ หรือเตรียม Stock ล่วงหน้า ซึ่งเป็นจุดขายสำคัญของระบบ

CRITICAL ไม่มี Dashboard ภาพรวมสำหรับทั้ง 2 ฝั่ง

ทั้ง JV Admin และ Hospital ไม่มีหน้า Dashboard ที่แสดงสถานะรวม เช่น จำนวน PO รอดำเนินการ, ยอดขายรวม, สินค้าที่ใกล้หมด Stock ทำให้ต้องเข้าไปดูทีละหน้า

CRITICAL ไม่มี Role สำหรับ Logistics / Warehouse

ระบบขาด Role สำหรับจัดการคลังสินค้าและการจัดส่ง ซึ่งจำเป็นต่อการทำงานจริง เช่น การรับ PO → จัดเตรียมสินค้า → ส่งออก → ยืนยันรับ

CRITICAL Flow การสร้าง User ผ่าน "ระบบภายนอก" ไม่ชัดเจน

ระบุว่าต้องติดต่อ Dev จัดทำ ซึ่งไม่ scalable และเป็นคอขวด ควรให้ JV Admin สามารถสร้าง User และจัดการ Role ได้เองผ่านระบบ

CRITICAL ไม่มี Notification System

ไม่มีระบบแจ้งเตือนเมื่อมี PO ใหม่, PO ถูก approve/reject, สินค้าจัดส่งแล้ว ฯลฯ ทำให้ต้อง login มาเช็คเอง

⚠️ ปัญหาที่ควรแก้ไข (UX Issues) 5 รายการ

UX ตารางข้อมูลยาวเกินไป ไม่มี Pagination / Filter ที่ดี

ตารางสินค้าและ PO Report ไม่มี search, filter ตามประเภทสินค้า, ช่วงวันที่ หรือ pagination ทำให้หา item ที่ต้องการลำบากเมื่อมีข้อมูลมาก

UX ขาดความสม่ำเสมอของ UI Pattern

หน้าจอแต่ละส่วนมี layout ที่แตกต่างกัน เช่น ปุ่ม action อยู่คนละตำแหน่ง, ตารางมีรูปแบบไม่เหมือนกัน ทำให้ผู้ใช้สับสน

UX ไม่มี Confirmation Dialog ก่อนทำ action สำคัญ

การ approve/reject PO ควรมี dialog ยืนยัน พร้อมช่องใส่เหตุผล (โดยเฉพาะ reject) เพื่อป้องกันการกดผิดและเก็บ audit trail

UX Mobile Responsive ไม่ชัดเจน

การออกแบบดูเหมือนเน้น Desktop เท่านั้น ทั้งที่ผู้ใช้ฝั่ง Hospital อาจใช้ Tablet หรือ Mobile ในการตรวจสอบสถานะ

UX ไม่มี Empty State / Error State

ไม่มีการออกแบบสำหรับกรณีที่ไม่มีข้อมูล หรือเกิด error ทำให้ผู้ใช้ไม่รู้ว่าต้องทำอะไรต่อ
